docs/ — documentação técnica que não cabe no README. Para o seu projeto você pode colocar:

docs/prompt.md — o prompt exato que você usa na LLM e por que escolheu aquele formato
docs/decisoes.md — registro das decisões de arquitetura (ex: "escolhi FastAPI porque...")
docs/idiomas.md — lista dos idiomas suportados com os códigos usados na API

Muita gente deixa vazia porque não sabe o que é "documentação técnica" na prática — mas qualquer explicação que ajude você (ou outra pessoa) a entender o projeto no futuro serve.
